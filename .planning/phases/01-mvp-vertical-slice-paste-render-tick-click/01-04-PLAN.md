# Plan 01-04: File Distribution and Phase 1 Smoke Verification

<objective>
Harden the Phase 1 vertical slice against the distribution constraints: one offline HTML file, no runtime network, no `file://`-blocked APIs, and smoke tests for parser/render/timer behavior.
</objective>

<context>
This is a closure plan, not a feature grab bag. It verifies the earlier plans did not accidentally add a build step, CDN dependency, module script, local fetch, or unreadable screenshare layout.
</context>

<tasks>
<task id="1" type="verification" files="meeting-manager.html">
<action>Audit the HTML for forbidden runtime patterns: external scripts/styles/fonts, `<script type="module">`, `fetch('agenda.md')` or other local fetches, package/build references, and non-system font dependencies.</action>
<verify>Search the file for `src=`, `href=`, `type="module"`, `fetch(`, `import `, `npm`, `package`, `fonts.googleapis`, and document any intentional false positives.</verify>
<acceptance_criteria>
- No runtime CDN dependency.
- No local-file fetch.
- No module script.
- System font stack is used.
</acceptance_criteria>
</task>

<task id="2" type="verification" files="meeting-manager.html">
<action>Run parser smoke fixtures, then paste the README agenda into the UI and verify the full path: load, title slide, agenda slides, sub-states, sidebar click, timer controls, and overrun banner.</action>
<verify>Use Chrome/Edge opened from disk. Record any manual verification notes in comments or a concise dev-only console summary.</verify>
<acceptance_criteria>
- No `NaN`, `undefined`, `[object Object]`, or blank stage.
- Fixture smoke test passes.
- README agenda renders correctly.
</acceptance_criteria>
</task>

<task id="3" type="verification" files="meeting-manager.html">
<action>Perform timer truth checks: short countdown crossing zero, pause/resume during positive time, reset during overrun, +/- adjustment, and brief background/foreground return.</action>
<verify>Compare expected elapsed wall time to displayed remaining time; visible display should snap to truth after foregrounding.</verify>
<acceptance_criteria>
- Overrun remains visible and negative.
- Reset returns to allocated duration and paused/known state.
- No timer drift pattern from missed ticks.
</acceptance_criteria>
</task>

<task id="4" type="verification" files="meeting-manager.html">
<action>Do a light layout/readability pass at 100%, 125%, and 150% browser zoom. Fix any overlap introduced in Phase 1 without pulling in Phase 4's full Discord audit scope.</action>
<verify>At all three zoom levels, sidebar, stage, and timer bar remain readable and non-overlapping.</verify>
<acceptance_criteria>
- No horizontal page scroll.
- Timer numerals stay stable.
- Stage content remains legible enough for the Phase 1 demo.
</acceptance_criteria>
</task>
</tasks>
