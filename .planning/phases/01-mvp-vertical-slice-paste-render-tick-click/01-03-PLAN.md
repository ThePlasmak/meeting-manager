# Plan 01-03: Timer Engine, Controls, and Overrun Visual

<objective>
Add truthful per-slide countdown behavior, sidebar click-to-jump-and-reset, timer controls, and the static overrun banner without introducing timer drift or per-slide intervals.
</objective>

<context>
The timer is the product's highest-risk behavior. It must derive elapsed time from `performance.now()` and accumulated durations, never from decrementing counters. Phase 1 can include minimal timer keyboard controls only where explicitly required by TIMER requirements; full navigation keyboard remains Phase 2.
</context>

<tasks>
<task id="1" type="implementation" files="agenda-presenter.html">
<action>Define timer state per timer key: `{ allocatedMs, accumulatedMs, startedAt, paused }`. Initialize title slide as no-timer and each agenda item/sub-state with its own independent timer object.</action>
<verify>Navigate between slides and inspect `state.timers`; each timer key remains independent.</verify>
<acceptance_criteria>
- Plain item keys use agenda/slide timer keys consistently.
- Per-person keys are composite strings such as `"3.0"`.
- Navigating away and back preserves timer state.
</acceptance_criteria>
</task>

<task id="2" type="implementation" files="agenda-presenter.html">
<action>Implement one `requestAnimationFrame` loop that renders timer truth from `performance.now()`. Add helpers `elapsedMs(timer)`, `remainingMs(timer)`, `pauseTimer`, `resumeTimer`, `resetTimer`, and `adjustTimer(deltaMs)`. Do not use `setInterval` to decrement counters.</action>
<verify>Start a short timer, background/alt-tab briefly, return, and confirm display snaps to truth instead of lagging by missed ticks.</verify>
<acceptance_criteria>
- Running state stores `startedAt = performance.now()`.
- Pausing folds elapsed time into `accumulatedMs` and clears `startedAt`.
- Remaining time can go negative.
</acceptance_criteria>
</task>

<task id="3" type="implementation" files="agenda-presenter.html">
<action>Render the timer bar with `MM:SS remaining / MM:SS allocated`, pause/play, reset, and +/- 1 minute controls. Use `font-variant-numeric: tabular-nums`. Add minimal guarded keyboard bindings for Space, `r`/`R`, `+`, and `-` only; full arrow/number navigation stays Phase 2.</action>
<verify>Buttons and minimal keys work when focus is not in a textarea/input; typing those keys in the paste textarea does nothing to timer state.</verify>
<acceptance_criteria>
- Timer color state is green at >=25%, amber below 25%, red at <=0.
- Pause/play does not create or destroy loops; it mutates timer state.
- Reset affects only the current timer key.
- +/- buttons and keys adjust current timer by 60 seconds.
</acceptance_criteria>
</task>

<task id="4" type="implementation" files="agenda-presenter.html">
<action>Implement sidebar click behavior for agenda rows and sub-items: jump to that slide/sub-state and reset the destination timer to its allocated duration. Preserve all other timer states.</action>
<verify>Run one timer partly down, click another sidebar item, then return; the original timer state is preserved while the clicked destination was reset.</verify>
<acceptance_criteria>
- Item-level click on a sub-state parent lands on first sub-state.
- Sub-item click lands on the named sub-state.
- Destination timer resets; unrelated timers do not.
</acceptance_criteria>
</task>

<task id="5" type="implementation" files="agenda-presenter.html">
<action>Add static overrun visual: when current timer remaining is <=0, tint the slide red and show a CSS-only `OVERTIME` banner with bold sans text, thick outline/stroke, slight skew, and shifted colors. Do not use external assets or animations.</action>
<verify>Set a timer to 5 seconds, let it cross zero, then pause/reset/advance. Banner stays while overrun, disappears on reset or non-overrun slide.</verify>
<acceptance_criteria>
- Whole slide tint is visible.
- Timer bar shows negative time with a minus prefix.
- Banner is static and CSS-only.
- User-facing copy calls it overrun/overtime behavior without using any external game assets.
</acceptance_criteria>
</task>
</tasks>

<verification>
- Start/pause/resume/reset current timer.
- Use +/- controls and confirm allocated/remaining display remains coherent.
- Let a timer cross zero and confirm overrun visual persists.
- Alt-tab test confirms no counter drift.
</verification>

<must_haves>
- No decrementing countdown counter.
- No per-slide interval lifecycle.
- Sidebar clicks reset destination only.
- Negative time is allowed and rendered.
</must_haves>

- SIDE-04, TIMER-01 through TIMER-06, TIMER-08 through TIMER-10, and OVER-01 through OVER-04 are satisfied.

</success_criteria>
